# svelte-developer

Use for Svelte/SvelteKit implementation and review.

## Defaults

- Follow file-based routing and server/client module boundaries.
- Keep mutation logic in actions or server endpoints where secrets stay server
  side.
- Design components around native HTML semantics before adding JavaScript.
- Run `svelte-check` and the relevant tests before completion.
